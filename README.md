Overview
The Limpopo Microloans Database System is a relational database designed to support the operations of a microfinance institution. The system manages clients, loans, repayments, penalties, and loan statuses while enforcing data integrity and business rules.
This project demonstrates practical SQL skills and structured database design applied to a financial services context.

Key Features
Client and loan record management
Loan repayment tracking
Penalty handling for overdue payments
Loan status updates after full repayment
Strong referential integrity using foreign keys

Technologies Used
MySQL
SQL (DDL & DML)
MySQL Workbench

Skills Demonstrated
Conceptual, logical, and physical data modelling
Database normalization
SQL table creation and constraints
Insert, update, delete, and select queries
Translating business rules into database logic

How to Run the Project
Open MySQL Workbench
Create a new SQL tab

Run the script files in this order:
create_database.sql
create_tables.sql
insert_data.sql
queries.sql
Use SELECT queries to verify data and relationships


Purpose

This system simulates a real-world microloan environment and highlights how structured databases support financial accuracy, accountability, and operational efficiency.

Notes

This project was developed as part of academic coursework and is intended to demonstrate database design and SQL proficiency.
